Introduction


Thrift Fund is an app that allows other people to donate items to be sold for your cause. In my implementation it works very close to the likes of Craigslist. The technology used behind the app is cordova for the front end and node.js for the backend (or API).

Running The App In XCode (IOS)


Running the app locally in XCode is super simple! Under ./buildIOS run Thift Fund.xcode then after Xcode opens click emulate selecting a target device (I recommend using a standard size iphone 7)

Since the app requires an API the computer must be able to connect to the internet, and able to navigate to thrift.fund

Running The App On Device (IOS)


I have uploaded the App to Apple's Test Flight Program which is used to allow beta testers to use an App. This is normally an invite only process since you need the testers email.

I have streamlined this process by creating a site which will take your email and add it to the allowed testers list. After you hit submit you should get an email from apple allowing you to download the app onto an IOS device.

I'm in no way collecting the email used, nor does the email have to be the same as your Apple Account email.

Compiling The App ADVANCED


Compiling the app anew is a more advanced process but can be easily achieved. The app is separating into two main components client and server.

Prerequisites: To run the app you must be on a OSX system with Node v7 and XCode installed.
After Node is installed run npm install -g bower grunt cordova

client/ contains everything needed to compile and run the front end of the app, the produced app files can be found in client/cordova/platform/ios. To compile the app on a unix system from the client/ directory run npm install && bower install && grunt compile after this is complete you can compile the IOS app iself by runnning cordova emulate ios --target="iPhone-7, 10.2" This should start up ios emulator and will only work on Mac OSX with XCode installed.

Server


Thrift Fund relies on a server to handle storing items, images, and users. In server/ you can see the code running the API found on thrift.fund. The server uses Mongodb as its database and JWT for user authentication. Sidenote if you are running locally and don't want to use the demo database you must modify the database connection string in server/config.js

The API has three main functions founds in server/services which are managing items, users, and users wishlists. The client is able to use the API routes from server/routes to update and retrieve data.
